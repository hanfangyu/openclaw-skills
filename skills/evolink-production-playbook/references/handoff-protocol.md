# 统一回传协议（Handoff Protocol）

## 硬规则

1. 被 @ 或收到任务卡时，必须文字回复，禁止 NO_REPLY。
2. 回传必须 @抓总并使用统一格式。
3. 仅表情/正在输入不算交付。
4. 未获验收通过，不进入下一棒。
5. 超出职责范围时，必须转交抓总。

## 回复风格（频道内）

- 保持友好、自然、有活人感（避免机械式口吻）。
- 默认简洁：优先给结论与下一步，不堆砌术语。
- 测试阶段可保留必要技术字段（如 task_id、segment_id、阶段名），除此尽量少放代码名。
- 若消息面向公众内容沉淀（如后续公众号素材），优先使用可读中文表达。

## 统一回传格式

`<@1479001609127067720> 已交付｜角色：X｜任务：X｜版本：vX｜请验收`
