# Language Map

把技术动作翻成故事语言时，优先保留“人”和“现场”。

## 常用替换

- `agent` -> 成员 / 同事 / 员工 / 角色
- `workflow` -> 接力 / 排练表 / 做事顺序
- `task card` -> 委托单 / 交代 / 今日任务
- `memory` -> 日记 / 旧档案 / 昨天留下的话
- `prompt` -> 口令 / 交代 / 台词
- `model` -> 手艺 / 天赋 / 擅长的路数
- `tool` -> 道具 / 手边的东西
- `dispatch` -> 派活 / 点将 / 叫上场
- `fallback` -> 兜底 / 补位
- `blocked` -> 卡住 / 堵住 / 没法往下走
- `deliverable` -> 交出来的东西 / 结果 / 成品

## 句式倾向

少写：

- 今天我们优化了流程
- 今天我们完成了配置
- 今天把协作链路跑通了

多写：

- 今天大家第一次知道，什么叫轮到自己开口
- 今天真正难的不是把事做完，而是不让场面散掉
- 今天有人接住了局面，也有人第一次露了短

## 禁忌

- 不要堆术语显得专业
- 不要像产品更新日志
- 不要像功能介绍
- 不要像教学笔记
